# CLAUDE.md — PROJECT MANA 運用ルール

## 変更前に必ず確認すること（2026-06-29 追加）

コードを変更してpushする前に、以下を自問する：

1. **この変更が壊すものは何か？** — 既存の動作のうち、影響を受けるものを列挙する
2. **モバイルとPCで挙動が変わるか？** — CSS変更はレスポンシブ両方を考慮する
3. **キャッシュの影響はあるか？** — 静的ファイル変更時はキャッシュを考慮する
4. **DBやAPIに依存する機能を変えたか？** — 代替手段が本当に同等か確認する

「直したらまた別が壊れた」は確認不足のサイン。1つの変更につき副作用まで責任を持つ。

## 基本原則：ユーザー制約（感情・配慮ではなくシステム要件）

**ユーザーはコード編集を前提としない。**

これは配慮ではない。システム要件である。

ユーザーは「こういうことになりたくなかったから何度もできないと言ってきた」。  
その言葉は感情表現ではなく、このシステムが守るべき制約の宣言である。

### 具体的な禁止事項

- コードの貼り付けを依頼しない
- GASエディタでの手作業を依頼しない
- 「ボタンを押してください」という指示を出さない
- 「Ctrl+Sで保存してください」という指示を出さない

### 代替手段を尽くす順序

1. 自分のツール（Drive MCP・GitHub MCP・Vercel MCP等）で解決する
2. 解決できない場合、その理由と代替アーキテクチャを提案する
3. それでも解決できない場合のみ、**1操作に絞って**依頼する
4. 依頼する場合は「これだけ」「画面のどこを」まで具体的に伝える

ユーザーにコード編集を依頼することは、代替手段を尽くした後の絶対的最終手段である。

### doGet に関するルール（2026-06-24 事故より）

- **doGet は `survey_receiver.gs` のものが正式。触らない。**
- `コード.gs`（gas_main.gs）に doGet を追加してはならない。
- GASプロジェクトに同名関数が複数存在すると実行エラーになる。
- GASを変更する前に必ず `download_file_content` で現在の全ファイルを確認する。

### GAS変更の手順

1. `mcp__Google_Drive__download_file_content` でプロジェクト全体を取得
2. 変更対象ファイルの現在の内容を確認
3. 既存の関数名と衝突しないか確認
4. **変更はユーザーに依頼せず、可能な手段を探す**
5. どうしてもユーザー操作が必要な場合は：
   - 1操作に絞る
   - 何をどこで押すか画面上の位置まで具体的に伝える
   - 「これだけ」と明示する

### Google Drive MCP の制限

- `download_file_content` — 読み取り可
- `create_file` — 新規作成のみ（既存ファイルの更新不可）
- GASファイルの直接更新ツールは現時点で存在しない

## セッション間の申し送りルール

### セッション開始時に必ずやること（順番通りに）

1. `docs/WORKLOG/` の最新ファイルを読む
2. 未完了タスクと前回の状態を把握してから作業開始
3. GASを触る場合は `download_file_content` で全ファイル確認

この手順を踏まずに作業を始めてはならない。

### セッション中・終了時

- 毎回 `docs/WORKLOG/YYYY-MM-DD.md` を作成・更新する
- 今日やったこと・未完了タスク・事故を記録してpushする
- 事故・失敗は必ず記録する（隠さない）
- 「できる」と申し送った内容が次セッションで再現できるか確認してから作業開始

## 2026-06-24 事故の記録

### 何が起きたか

1. `コード.gs` に `doGet` 関数を追記した
2. すでに `survey_receiver.gs` に doGet が存在していた
3. GASプロジェクトに同名関数が複数になり、実行エラー（403）が発生
4. 観測DB（410件）が表示されなくなった
5. 復旧のためにユーザーに複数回の手作業を依頼してしまった

### 根本原因

- GASプロジェクト全体の構造を確認せずに関数を追加した
- 「doGetがない」と判断したのはGitHubのgas_main.gsだけを見たから
- 実際のGASプロジェクトには複数のファイルがある

### 再発防止

- GAS変更前は必ず全ファイルを `download_file_content` で確認する
- 同名関数がないか必ず検索してから追加する
- doGet は `survey_receiver.gs` のものを正式とし、他のファイルには書かない
