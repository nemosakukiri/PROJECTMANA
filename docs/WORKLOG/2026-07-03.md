### PWA対応（コミット `0e223ec`）
- 目的：スマホで「ホーム画面に追加」→ MANAロゴアイコンからアプリのように起動。
- 追加ファイル：
  - `manifest.json`（name=PROJECT MANA / short_name=MANA / display=standalone / theme_color=#faf9f6 / background_color=#ece2bc）
  - `icon-512.png` `icon-192.png` `apple-touch-icon.png`（**logo.svgの図形をPillowで再現**して生成。パーチメント背景 #ece2bc に MANAロゴ）
  - `sw.js`（Service Worker。**network-first**＝オンライン時は常に最新／同一オリジンGETのみ扱い、GAS API・フォント等の外部通信には干渉しない。キャッシュ事故を起こしにくい設計）
  - `pwa.js`（インストール案内バナー：Android=beforeinstallprompt→「追加」ボタン／iOS Safari=共有→ホーム画面に追加の手順案内。一度✕で閉じたら14日間は再表示しない＝localStorage `mana_pwa_dismissed_at`）
- `index.html` head に manifest・theme-color・apple-touch-icon・apple用メタを追加、末尾に `<script src="/pwa.js" defer>`。
- vercel.json は `filesystem` 優先配信のため manifest/sw/アイコンはそのまま配信（設定変更不要）。
- **ユーザー環境（Mac Chrome）でインストール成功を確認済み**（Chromeアプリに「PROJECT MANA」アイコンが追加された）。
- アイコン生成にPillowを使用（`pip install pillow`）。この環境にはSVG→PNG変換ツール（rsvg/convert/inkscape/cairosvg）が無かったため、図形を直接描画して生成した。

### 収集ソース追加：論考のシノドス偏重を是正＋社説強化（GAS `コード.gs`）
- 背景：論考の家（研究者・article_type=research）が**シノドスばかり**で偏っていた。多様化したい＋社説をもっと厚く。
- `RSS_SOURCES` に6ソース追加（既存の「Google News RSS × `site:ドメイン`」方式に合わせた）：
  - research（論考の家）：東京財団政策研究所(tkfd.or.jp)／社会構想デザイン機構ISVD(isvd.or.jp)／朝日Re:Ron(asahi.com)
  - opinion：日経グローバルオピニオン(nikkei.com)／社説・論説（横断）／東京新聞 社説(tokyo-np.co.jp)
- **本番反映方法（重要・再現用）**：clasp認証が「Invalid Credentials」で失敗 → `~/.clasprc.json` の refresh_token で `https://oauth2.googleapis.com/token` から access_token を再取得 → **Apps Script API `PUT /v1/projects/{scriptId}/content`** で全ファイルPUT（`コード`のsourceだけ差し替え、他ファイルは取得したものをそのまま戻す）。反映成功（13ファイル）。
- 効き方：**次回 collectNews 実行時から**収集・分類されて各家に出る（即時ではない）。朝日/日経は有料の壁で見出し＋リンクまで。
- 注意：GASはGitHubからデプロイされない。`コード.gs`をGitHubにcommitしても本番は変わらない。**本番反映はApps Script APIのPUTが必須**（上記手順）。サンドボックスからGoogle NewsへはHTTP不可(403)だがGAS本体はGoogle上で動くので問題なし。
