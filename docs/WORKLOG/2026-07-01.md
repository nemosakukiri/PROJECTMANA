## スマホ対応の実装設計（次セッションで即実装できるよう詳細を残す）

対象ファイル: `ehon-no-kurashi/index.html`（本番は独立リポジトリ `ehon-no-kurashi` のルート `index.html`）

### 方針
- PC（広い画面）: 現状維持（左に `#log-panel` 常時表示）。
- スマホ幅（`max-width: 640px`）: `#log-panel` をデフォルトで画面外へスライドアウトし、
  トップバーのトグルボタンで開閉する。開いている間は広場の上にオーバーレイ表示。

### CSS 追加案（`</style>` の直前あたりに追記）
```css
/* スマホ：つぶやきパネルをスライド開閉に */
#log-toggle { display: none; }
@media (max-width: 640px) {
  #log-toggle {
    display: inline-block; margin-left: 8px;
    background: rgba(80,140,80,0.5); border: 1px solid rgba(160,210,160,0.4);
    border-radius: 8px; padding: 4px 10px; color: #fff; font-size: 12px;
    cursor: pointer; font-family: inherit;
  }
  #log-panel {
    width: 78%; max-width: 300px;
    transform: translateX(-100%);
    transition: transform 0.28s ease;
    z-index: 120;
    box-shadow: 2px 0 16px rgba(0,0,0,0.35);
  }
  #log-panel.open { transform: translateX(0); }
}
```

### HTML 変更案
- トップバー（`#top-bar`）に開閉ボタンを追加:
```html
<span class="logo">絵本のくらし</span>
<button id="log-toggle" onclick="toggleLogPanel()">💬 つぶやき</button>
<span id="online-count">今ここにいる人 ― 人</span>
```

### JS 追加案（★グローバルスコープに置くこと。6/28の教訓：DOMContentLoaded内だとonclickから呼べない）
```js
function toggleLogPanel() {
  const panel = document.getElementById('log-panel');
  const open = panel.classList.toggle('open');
  document.getElementById('log-toggle').textContent = open ? '✕ 閉じる' : '💬 つぶやき';
}
```
- 補足: `enterHiroba()` で `#log-panel` を `display:flex` にした後も、
  スマホでは `transform` で画面外に退避しているので初期状態は隠れる（displayは変更しない）。
- 任意: 広場（`#scene`）タップで開いているパネルを閉じる処理を足すとより自然。
